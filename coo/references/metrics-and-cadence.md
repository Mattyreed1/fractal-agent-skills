# Metrics & Cadence

Deep reference for the COO skill's metrics and operational rhythm.

## Leading vs Lagging Indicators

| Type | Examples | Purpose |
|------|----------|---------|
| **Lagging** | Revenue, churn, profit | Confirm damage (too late to steer) |
| **Leading** | Cycle time, defect rate, WIP, on-time delivery, rework rate | Steer with these |

COOs steer with leading indicators. Lagging metrics only confirm damage.

## Minimum Viable Dashboard

| Metric | What It Measures |
|--------|-----------------|
| **Throughput** | Units of value delivered per time period |
| **Cycle time** | How long work takes from start to done |
| **Quality yield** | Percentage correct on first pass |
| **Forecast vs actual** | Drift between committed and delivered on time and cost |
| **Burn/runway or margin** | How fast cash moves and how much room remains |
| **Context graph integrity** | Whether the system's model of reality matches actual state |

## The COO Cadence

### Daily
- What shipped
- What broke
- What's blocked

### Weekly
- Bottleneck review
- Commitments for next week
- System changes (not just tasks)

### Monthly
- Capacity vs demand
- Error budgets
- Quality trends
- Cost vs value

### Quarterly
- Strategy → policy updates
- Kill low-leverage work
- Redesign risk envelopes

## What Good Looks Like

A mature ops system:
- Detects problems early
- Limits damage automatically
- Improves itself through feedback
- Escalates risk before disaster
- Makes reality visible without meetings

### The Test
If you leave for two weeks and:
- Work still flows
- Risk stays bounded
- Truth stays visible
- Quality stays high

You built a system that runs itself. That's operational excellence.
