# MCP Server Setup Skill

Expert guide for setting up MCP servers on this machine. **Default environment is Claude Desktop (the app).**

## CRITICAL: Know Your Environment

| Environment | Config Location | How to Add Servers |
|-------------|-----------------|-------------------|
| **Claude Desktop (DEFAULT)** | `~/Library/Application Support/Claude/claude_desktop_config.json` | Edit JSON directly |
| Claude Code CLI | `~/.claude.json` | Use `claude mcp add` commands |

**ALWAYS assume Claude Desktop unless user explicitly says "CLI" or "terminal".**

---

## Quick Decision Tree

```
Adding an MCP server?
│
├─ Is it HTTP/OAuth? (URL starts with https://)
│   ├─ YES → Use mcp-remote bridge (Section A)
│   └─ NO → Use direct Node execution (Section B)
│
└─ What type of package?
    ├─ npm package → Install to ~/.local, run with Node 20
    └─ Remote URL → Use mcp-remote
```

---

## Section A: HTTP/OAuth MCP Servers (Notion, Canva, etc.)

These servers use OAuth and have URLs like `https://mcp.service.com/mcp`.

### Step 1: Install mcp-remote (one-time)

```bash
/usr/local/opt/node@20/bin/npm install --prefix ~/.local mcp-remote
```

### Step 2: Add to Claude Desktop Config

Edit `~/Library/Application Support/Claude/claude_desktop_config.json`:

```json
{
  "mcpServers": {
    "service-name": {
      "command": "/usr/local/opt/node@20/bin/node",
      "args": [
        "~/.local/node_modules/mcp-remote/dist/proxy.js",
        "https://mcp.service.com/mcp"
      ]
    }
  }
}
```

### Step 3: Authenticate

1. Quit Claude Desktop completely (Cmd+Q)
2. Reopen Claude Desktop
3. Try to use a tool from that server
4. Browser opens for OAuth - authorize the app
5. Done!

### Known HTTP/OAuth Servers

| Service | URL | Config Name |
|---------|-----|-------------|
| Notion | `https://mcp.notion.com/mcp` | `mr-notion` |
| Canva | `https://mcp.canva.com/mcp` | `canva` |

## Section B: npm Package MCP Servers

These are traditional stdio servers distributed as npm packages.

### Step 1: Choose a Short Name

- Max 20 characters
- kebab-case: `my-server` not `myServer`
- Prefix: `mr-` (personal), `scaleagency-` (work)

### Step 2: Install the Package

```bash
/usr/local/opt/node@20/bin/npm install --prefix ~/.local <package-name>
```

### Step 3: Find the Entry Point

```bash
cat ~/.local/node_modules/<package-name>/package.json | grep -A5 '"bin"'
```

Common patterns:
- `dist/index.js`
- `build/index.js`
- `dist/mcp/stdio-wrapper.js`

### Step 4: Add to Claude Desktop Config

```json
{
  "mcpServers": {
    "server-name": {
      "command": "/usr/local/opt/node@20/bin/node",
      "args": [
        "~/.local/node_modules/<package-name>/<entry-point>.js"
      ],
      "env": {
        "API_KEY": "your-key-here"
      }
    }
  }
}
```

### Step 5: Restart Claude Desktop

Quit completely (Cmd+Q) and reopen.

## Section C: The 64-Character Limit

Claude prefixes tool names as `mcp__<server-name>__<tool-name>`. API limit is 64 chars.

**BAD:**
```
mcp__<auto-generated-uuid>__notion-query-database-view
     └── 36 char UUID ──────────────────┘
Total: 68 chars = API ERROR
```

**GOOD:**
```
mcp__mr-notion__notion-query-database-view
     └─ 9 chars ─┘
Total: 43 chars = Works!
```

## Section D: This Machine's Node.js Issue

**PROBLEM:** nvm installs multiple Node versions. Claude Desktop inherits PATH with old Node v16 first. Using `npx` fails because child processes inherit the broken PATH.

**SOLUTION:** Never use `npx` in Claude Desktop config. Always:
1. Install with: `/usr/local/opt/node@20/bin/npm install --prefix ~/.local <pkg>`
2. Run with: `/usr/local/opt/node@20/bin/node <path-to-entry.js>`

**Working Node paths:**
```
/usr/local/opt/node@20/bin/node
/usr/local/opt/node@20/bin/npm
/usr/local/opt/node@20/bin/npx  # Only use in terminal, NOT in config
```

## Section E: Debugging

Deep debugging walkthrough: references/debugging.md

### Check Logs
```bash
tail -100 ~/Library/Logs/Claude/mcp-server-<name>.log
tail -100 ~/Library/Logs/Claude/mcp*.log
```

### Test Server Manually
```bash
API_KEY="value" /usr/local/opt/node@20/bin/node ~/.local/node_modules/<pkg>/dist/index.js
```

### Common Errors

| Error | Cause | Fix |
|-------|-------|-----|
| `tool_name: String should have at most 64 characters` | Server name too long | Use shorter name (<20 chars) |
| `npm does not support Node.js v16` | PATH issue | Use `/usr/local/opt/node@20/bin/node` |
| `Class extends value undefined` | Wrong Node version | Reinstall with Node 20 |
| `Server disconnected` | Crash on startup | Check logs for real error |
| `ENOENT` | Wrong file path | Verify path exists |

## Section G: Adding a New Server Checklist

```
□ 1. Determine type: HTTP/OAuth or npm package?
□ 2. Choose short name (<20 chars, kebab-case)
□ 3. If npm: Install with Node 20's npm to ~/.local
□ 4. If npm: Find entry point in package.json "bin" field
□ 5. If HTTP: Ensure mcp-remote is installed
□ 6. Edit ~/Library/Application Support/Claude/claude_desktop_config.json
□ 7. Use /usr/local/opt/node@20/bin/node as command (NEVER npx)
□ 8. Quit Claude Desktop completely (Cmd+Q)
□ 9. Reopen Claude Desktop
□ 10. If HTTP/OAuth: Authorize in browser when prompted
□ 11. Test by asking to use a tool from that server
□ 12. If fails: Check ~/Library/Logs/Claude/mcp-server-<name>.log
```

## Section H: Google APIs Setup Reference

### Google Calendar

1. Go to console.cloud.google.com
2. Create project, enable Google Calendar API
3. Configure OAuth consent screen (add yourself as test user!)
4. Create OAuth credentials (Desktop app)
5. Download JSON to `~/.config/google-calendar-mcp/credentials.json`
6. Authenticate:
   ```bash
   GOOGLE_OAUTH_CREDENTIALS="$HOME/.config/google-calendar-mcp/credentials.json" \
     npx -y @cocal/google-calendar-mcp auth
   ```

### Google Maps

1. Go to console.cloud.google.com
2. Enable: Places API (New), Geocoding API, Routes API
3. Create API Key (APIs & Services > Credentials)
4. Add key to config env vars

## Section I: n8n-mcp Server Setup (CRITICAL)

The `n8n-mcp` package provides two tiers of functionality:

| Tier | Tools Available | Requirements |
|------|-----------------|--------------|
| **Reference only** | `search_nodes`, `get_node`, `validate_node`, `get_template`, `search_templates`, `validate_workflow` | None |
| **Full API access** | Above + `n8n_create_workflow`, `n8n_get_workflow`, `n8n_update_*`, `n8n_delete_workflow`, `n8n_list_workflows`, `n8n_test_workflow`, `n8n_executions`, etc. | `N8N_API_URL` and `N8N_API_KEY` env vars |

### PROBLEM: Missing API Tools

If you can search nodes but CANNOT create/update/list workflows, the n8n-mcp server is missing its API credentials.

**How to check:** Look for `n8n_create_workflow` in available tools. If missing, credentials are not configured.

### SOLUTION: Add Environment Variables

**mr-n8n (personal n8n instance):**
```json
"mr-n8n": {
  "command": "/usr/local/opt/node@20/bin/node",
  "args": [
    "~/.npm/_npx/f34eeb202cab04b9/node_modules/n8n-mcp/dist/mcp/stdio-wrapper.js"
  ],
  "env": {
    "N8N_API_URL": "https://mr-automations.app.n8n.cloud",
    "N8N_API_KEY": "<your-api-key>"
  }
}
```

⚠️ This path lives in the npx cache and is destroyed by cache clears (including the cache-clear step in references/debugging.md). Prefer installing n8n-mcp under ~/.local (Section B) and pointing the config there.

### How to Get n8n API Key

1. Log into your n8n instance
2. Go to Settings > API
3. Create a new API key
4. Copy the key (starts with `eyJ...`)

### Multiple n8n Instances

You can connect to multiple n8n instances - each MCP server entry just needs its own URL and API key:

```json
"mr-n8n": {
  "command": "/usr/local/opt/node@20/bin/node",
  "args": ["..."],
  "env": {
    "N8N_API_URL": "https://mr-automations.app.n8n.cloud",
    "N8N_API_KEY": "key-for-personal"
  }
},
"work-n8n": {
  "command": "/usr/local/opt/node@20/bin/node",
  "args": ["..."],
  "env": {
    "N8N_API_URL": "https://work.app.n8n.cloud",
    "N8N_API_KEY": "key-for-work"
  }
}
```

### Troubleshooting n8n-mcp

| Symptom | Cause | Fix |
|---------|-------|-----|
| Only 7 tools available (search/validate only) | Missing `N8N_API_URL` and `N8N_API_KEY` | Add env vars to config |
| API tools exist but fail | Wrong URL or invalid key | Verify URL ends without `/`, key is valid |
| "Unauthorized" errors | API key expired or revoked | Generate new key in n8n Settings > API |

## Section J: Diagnosing "Missing Tools" Issues

When an MCP server seems to be missing tools:

### Step 1: Check What Tools Are Available

Use `ToolSearch` with the server prefix to see loaded tools:
```
ToolSearch: "+mr-n8n workflow"
```

### Step 2: Compare Against Expected Tools

Check the MCP server's documentation for the full tool list. For n8n-mcp:
- Call `mcp__mr-n8n__tools_documentation` to see what SHOULD be available
- If docs mention tools that aren't showing, config is incomplete

### Step 3: Read the Config File

```bash
cat ~/Library/Application\ Support/Claude/claude_desktop_config.json
```

Look for:
- Missing `env` block
- Missing required environment variables
- Typos in env var names

### Step 4: Fix and Restart

1. Edit the config to add missing env vars
2. Quit Claude Desktop completely (Cmd+Q)
3. Reopen Claude Desktop
4. Verify tools are now available
